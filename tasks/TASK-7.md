# TASK-7 — Этап 7: поиск-чат

> Результат этапа: жена у кабинета врача спрашивает своими словами — «когда Арине делали манту?» — и получает ответ, собранный строго из записей карты, с обязательными ссылками на записи-источники; тап по источнику открывает карточку. Нет опоры в записях — честное «в карте этого не нашлось».
>
> Источники истины: **`docs/flows/search.md`** (утверждён 19.07.2026, решения ❓1–10) · **ADR-018** (эмбеддинги через RouterAI, хранение, журнал тюнинга) · ADR-012 (в поиске только подтверждённое) · ADR-013 (паттерн AI-адаптера).
>
> Замечание к ❓8: в этапе экран показывает текущую пару «вопрос-ответ» (каждый вопрос самостоятелен, переписка в БД не хранится); лента прошлых пар сессии — бэклог, если понадобится.

---

### T7.1-BE — Слой эмбеддингов и хранилище векторов
Блокирует: T7.2, T7.3
Заблокирован: —

**Цель:** запись и запрос можно превратить в вектор; векторам есть где жить и как искаться.

**Scope:**
- Конфиг: `EMBEDDINGS_PROVIDER` (`openai_compatible` | `disabled`, дефолт `disabled`) + `EMBEDDINGS_BASE_URL/MODEL/API_KEY` (валидация полноты — паттерн экстрактора); `SEARCH_TOP_K` (дефолт 5), `SEARCH_MAX_DISTANCE` (порог «не нашёл», дефолт подобрать).
- Сверка каталога RouterAI: выбрать embedding-модель по критериям ADR-018 (русский язык, dim ≤1536), **дописать модель и размерность в ADR-018**.
- `services/embeddings.py`: Protocol (`embed(texts: list[str]) -> list[list[float]]`), адаптер OpenAI-совместимого `/embeddings` (httpx, без SDK), фабрика по конфигу, доменные ошибки.
- Миграция: `CREATE EXTENSION IF NOT EXISTS vector` · `record_embeddings` (record_id PK/FK, `embedding vector(dim)`, `model`, `updated_at`) · `search_queries` (id, question, candidates JSONB — [{record_id, distance}], answer TEXT NULL, created_at). ANN-индекс не создаём (ADR-018: точный поиск на семейных объёмах).
- `repositories/embeddings.py`: `upsert(record_id, vector, model)` · `delete_for_record(record_id)` · `search_similar(session, family_id, vector, k, max_distance) -> [(HealthRecord, distance)]` — join к записям, **фильтры по умолчанию: подтверждённые, неудалённые, своя семья**; `log_query(...)` для журнала.

**Явно НЕ входит:** индексация в жизненном цикле (T7.2), чат (T7.3), UI (T7.4).

**Контракт:** сигнатуры выше; вектор запроса и записи — одна модель (из конфига).

**Критерии приёмки:**
- Адаптер (MockTransport, без сети): happy-path, батч, HTTP-ошибка → доменная, `disabled` → фабрика отдаёт None/ошибку по образцу экстрактора.
- Интеграционно на pgvector: upsert + повторный upsert (перезапись), search_similar находит ближайшее, **не находит** неподтверждённые/удалённые/чужой семьи, порог отсекает далёкое.
- Миграция up→down→up чистая; ADR-018 дополнен конкретной моделью.

### T7.2-BE — Индексация в жизненном цикле записи
Блокирует: T7.3
Заблокирован: T7.1

**Цель:** индекс сам остаётся актуальным: подтверждение и правка индексируют, удаление вычищает; существующие записи догоняются инструментом.

**Scope:**
- `services/indexing.py`: `build_index_text(record)` — конкатенация по ❓5 (пациент, название, тип, клиника, врач, дата события, содержание, комментарий); `index_record(record_id, settings, session_factory)` — фоновая задача (паттерн `run_extraction`): эмбеддинг → upsert; провал → warning-лог, ничего не блокирует (B7).
- Вызовы: `POST /records/{id}/confirm` — после commit'а BackgroundTasks (первичное подтверждение и правка — один и тот же confirm); `soft_delete` → `delete_for_record` (синхронно, в той же транзакции).
- `app/tools/reindex.py`: бэкфилл всех подтверждённых неудалённых записей (идемпотентен; печатает счётчик; пригоден для смены модели).

**Явно НЕ входит:** ретроактивная индексация при ретрае конвейера (confirm — единственный триггер, B4 потока).

**Критерии приёмки:**
- Confirm (первичный и повторный) с fake-провайдером кладёт/обновляет вектор; текст индекса собирается по ❓5 (юнит).
- `soft_delete` удаляет строку эмбеддинга; подтверждение при `EMBEDDINGS_PROVIDER=disabled` или упавшем провайдере проходит как ни в чём не бывало (warning в логе — проверяется).
- `reindex` доиндексирует пропущенное и не плодит дублей; неподтверждённые не индексируются.

### T7.3-BE — Чат: retrieval + генерация с источниками + журнал
Блокирует: T7.4
Заблокирован: T7.1, T7.2

**Цель:** вопрос словами превращается в ответ строго по записям с источниками — или в честное «не нашлось»; каждый вопрос попадает в журнал тюнинга.

**Scope:**
- `services/llm.py`: минимальный chat-completion хелпер (httpx) на конфиге экстрактора (❓9 — тот же RouterAI/модель/ключ); JSON-контракт ответа: `{answer: str|null, source_ids: [int]}` (паттерн экстрактора: ретрай при невалидном JSON).
- `services/search_chat.py` — `ask(session, settings, family_id, question) -> SearchResult`:
  1. пустой вопрос → без обращений (B1);
  2. эмбеддинг вопроса; провайдер недоступен → `SearchResult(unavailable)` (B3);
  3. `search_similar` топ-K с порогом; пусто → `not_found` (B2, LLM не дёргается);
  4. LLM: system-промпт «отвечай ТОЛЬКО фактами из выдержек; нет ответа — верни null; перечисли id использованных записей»; `source_ids` фильтруются по кандидатам (галлюцинированные id отбрасываются);
  5. `answer` без валидных `source_ids` → `not_found_with_similar` (B5/B6: ответ без источников не существует); LLM упал → `llm_unavailable` + кандидаты (B4 — деградация до списка);
  6. журнал: вопрос, кандидаты с дистанциями, итоговый ответ (❓10).
- Роуты `routes/search.py`: `GET /search` (пустой чат), `POST /search` (form `question`) → рендер `search.html` с `SearchResult`. Доступ — сессия (default-deny сам закроет).

**Явно НЕ входит:** вёрстка (T7.4), диалоговый контекст (❓8), стриминг ответа (бэклог).

**Контракт:** контекст `search.html`: `question`, `result.kind` (`answer | not_found | not_found_with_similar | unavailable | llm_unavailable`), `result.answer`, `result.sources: [{record: поля rec-lite + patient_name + accent, distance}]`, `result.similar` (для деградаций).

**Критерии приёмки:**
- Fake-провайдеры: happy-path (ответ + источники ⊆ кандидатов), галлюцинированный id отброшен, ответ без источников → not_found_with_similar, пусто в retrieval → not_found без вызова LLM, LLM-ошибка → llm_unavailable с кандидатами, эмбеддинг-ошибка → unavailable.
- Каждый непустой вопрос — строка в `search_queries` (включая «не нашлось»).
- Чужая семья в выдачу не попадает (интеграционный).

### T7.4-FE — Экран чата и плавающая кнопка
Блокирует: —
Заблокирован: T7.3

**Цель:** поиск выглядит как чат и открывается одним тапом с ленты, не мешая записям.

**Scope:**
- Кит (§8, новые примитивы с обоснованием): **плавающая кнопка** `fab` («Найти» — компактная, текстовая, правый низ, выше `action-bar`, не перекрывает контент); **реплики чата** `chat-q` (вопрос — «карандашная» реплика оператора) / `chat-a` (ответ — «чернильный» лист); блок источников — заголовок-подпись + список `rec-lite` (переиспользование). Затем DESIGN §4, CSS, шаблон.
- `search.html`: приглашение (пустое состояние) · пара «вопрос-ответ» текущего запроса · источники под ответом · тексты B2/B3/B4 из потока (говорят, что делать) · форма вопроса закреплена снизу (`action-bar`-семейство, «Спросить» — primary этого экрана) · «‹ В ленту».
- `index.html`: плавающая «Найти» → `/search` (на пустом состоянии ленты кнопки нет — искать не в чем).
- Источник — `rec-lite` с монограммой/именем пациента (❓3): вариант компакт-элемента с акцентом пациента.

**Примитивы кита:** `fab`, `chat-q`/`chat-a`, `rec-lite`, `action-bar`, `empty`.

**Критерии приёмки:**
- Чек-лист DESIGN §9: одна primary на экран («Спросить»; «Найти» на ленте — не primary), только токены, тексты ошибок говорят что делать, пустые состояния есть.
- Тесты рендера: приглашение · ответ с источниками (и именами пациентов) · «не нашлось» · деградация B4 (список без ответа) · «поиск недоступен» · кнопка «Найти» на ленте с записями и её отсутствие на пустой ленте.
- Ручная проверка на телефоне: кнопка не перекрывает последнюю запись и `action-bar`; чат читается; тап по источнику → карточка.

## Порядок работы

```
T7.1-BE ── T7.2-BE ── T7.3-BE ── T7.4-FE
```

Этап закрыт, когда: с ленты одним тапом открывается чат; вопрос своими словами возвращает ответ с обязательными источниками-записями (тап → карточка); «не нашлось» честное; неподтверждённое/удалённое/чужое не ищется никогда; подтверждение/правка/удаление сами поддерживают индекс; `reindex` догоняет существующие записи; каждый вопрос — в журнале тюнинга; провал любого AI-провайдера деградирует поиск, не задевая остальной продукт; тесты и стражи зелёные.
