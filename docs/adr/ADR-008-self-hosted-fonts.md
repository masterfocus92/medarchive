# ADR-008 — Шрифты self-hosted, не с CDN

## Статус

Принято владельцем 13.07.2026 (одобрение плана T1.4-FE).

## Контекст

Дизайн-система использует Source Serif 4, Inter и JetBrains Mono. Стандартный путь — Google Fonts CDN — означает: первая отрисовка зависит от внешнего хоста (продукт открывают с телефона в клинике, сеть плохая), каждый визит утекает третьей стороне (меддокументы — чувствительный контекст), а self-host VPS остаётся зависимым от чужой инфраструктуры.

## Решение

Файлы woff2 лежат в `app/static/fonts/` и отдаются самим приложением; `@font-face` — в `app/static/css/app.css`. Скачаны с Fontsource (сборки Google Fonts), по два подмножества на начертание — latin и cyrillic с `unicode-range`: браузер забирает только нужное. `font-display: swap` — текст виден системным шрифтом до загрузки. Набор начертаний минимальный, по ролям системы: Inter 400/500, Source Serif 4 600 и 400 italic (карандаш), JetBrains Mono 400.

В рантайме продукт не делает ни одного внешнего запроса за статикой.

## Последствия

- (+) Первая отрисовка не зависит от внешних CDN; ноль утечек визитов.
- (−) ~160 КБ бинарников в git; обновление шрифтов — ручное (скачать новые файлы), автообновлений нет.
- Новое начертание (вес/стиль) — это новые файлы обоих подмножеств + пара @font-face, не строчка в `<link>`.
