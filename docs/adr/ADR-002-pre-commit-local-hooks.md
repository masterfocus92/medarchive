# ADR-002 — Pre-commit: локальные хуки через `uv run ruff`

## Статус

Принято владельцем 13.07.2026 (одобрение плана T1.1-INFRA, приёмка реализации).

## Контекст

Ruff закреплён в dev-зависимостях (`uv.lock`). Стандартный путь — репозиторий `ruff-pre-commit` с собственным `rev` — создаёт вторую, независимую версию линтера: возможна ситуация «pre-commit ругается, а `uv run ruff` — нет» (и наоборот), плюс отдельный процесс обновления rev.

## Решение

Хуки в `.pre-commit-config.yaml` объявлены локальными (`repo: local`, `language: system`) и вызывают `uv run ruff check --fix` и `uv run ruff format`. Версия линтера всегда одна — из `uv.lock`.

## Последствия

- (+) Единственный источник версии Ruff; обновление — обычный `uv` -путь.
- (−) Хуки требуют установленного `uv` на машине (для этого проекта — данность).
- (−) `pre-commit autoupdate` не работает — обновление версии только через `uv`.
- Pre-commit видит только файлы, известные git (untracked до `git add` не проверяются) — для полного прогона нужен хотя бы стейджинг.
