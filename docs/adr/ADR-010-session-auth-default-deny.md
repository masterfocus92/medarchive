# ADR-010 — Сессия в подписанной cookie; защита приложения — default-deny

## Статус

Принято владельцем 13.07.2026 (одобрение плана T2.2-BE).

## Контекст

По спеке вход — одноразовый акт на устройство: долгоживущая сессия, две предзаведённые учётки, без восстановления пароля. Инвариант: файлы и записи недоступны без аутентификации, включая прямые ссылки. Нужен механизм сессии и способ гарантировать, что ни один будущий роут не окажется случайно открытым.

## Решение

**Сессия** — `SessionMiddleware` Starlette: подписанная (`itsdangerous`) cookie, `max_age` 180 дней, `httponly`, `samesite=lax`. Секрет подписи — `Settings.secret_key` (обязательный, ≥ 32 символов — иначе приложение не стартует; в git не попадает). Серверного хранилища сессий нет — при двух операторах отзыв сессий не нужен; смена секрета разлогинивает всех.

**Защита — default-deny middleware** (`app/middleware.py`): всё, что не в явном списке публичного (`/login`, `/health`, `/static/*`), без сессии получает 303 на `/login`. Роут защищён самим фактом существования; публичность — сознательное расширение списка. Файлы записей будут раздаваться не из `/static`, а отдельным защищённым роутом (этап 3+) — на них default-deny действует автоматически.

**Против оракула email:** ответы (и время — через bcrypt-проверку фиктивного хэша) при «нет такого email» и «неверный пароль» неразличимы.

## Последствия

- (+) «Забыть про auth» на новом роуте невозможно; сессия не требует таблицы и чистки.
- (−) `https_only=False` — **обязательство этапа 8**: за HTTPS на VPS включить `https_only=True` (иначе cookie утекает по http).
- (−) Подписанная cookie не отзывается точечно — только сменой секрета (для семьи из двух операторов приемлемо).
- Точка входа — только factory-режим (`uvicorn app.main:create_app --factory`): сборка приложения требует настроек, импорт модуля — нет.
