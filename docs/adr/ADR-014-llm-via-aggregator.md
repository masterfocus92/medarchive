# ADR-014 — Доступ к LLM через OpenAI-совместимого посредника (RouterAI)

## Статус

Принято владельцем 14.07.2026.

## Контекст

Оплата официального Anthropic API из региона владельца недоступна (подписка Claude Max на API-кредиты не конвертируется). Российские агрегаторы-реселлеры дают доступ к тем же моделям с локальной оплатой через OpenAI-совместимый API. Владелец выбрал **RouterAI** (routerai.ru), ключ получен.

Факты из документации RouterAI (проверено 14.07.2026):
- База: `https://routerai.ru/api/v1`, аутентификация `Authorization: Bearer <ключ>`, формат OpenAI (`/chat/completions`).
- Claude доступен как **`anthropic/claude-sonnet-5`** (vision: да; 1M контекст; ~199₽/996₽ за 1M токенов).
- Изображения: `image_url` с base64 data-URI; форматы **png/jpeg/webp/gif** — HEIC и PDF напрямую не принимаются.
- Embeddings-модели в каталоге есть (27 шт.) — кандидат для Э7 при той же проблеме оплаты Voyage.

## Решение

1. Первый рабочий адаптер экстрактора — **универсальный OpenAI-совместимый** (`EXTRACTOR_PROVIDER=openai_compatible`): конфиг `EXTRACTOR_BASE_URL`, `EXTRACTOR_MODEL`, `EXTRACTOR_API_KEY`. Подходит для любого агрегатора; RouterAI — первое использование. HTTP — через `httpx` (уже в зависимостях транзитивно от anthropic SDK), без установки openai-SDK.
2. Конвертация форматов — **на нашей стороне**, в отдельном сервисе подготовки изображений: HEIC → JPEG (обязательство ❓8), PDF → PNG постранично (пригодится и для миниатюр Э5).
3. Родной адаптер Anthropic API (`EXTRACTOR_PROVIDER=claude`) — в бэклог: включается сменой конфига, когда появится официальный ключ. ADR-013 (Protocol + DTO) действует без изменений — ровно он делает эту развилку дешёвой.

## Размен приватности — принят владельцем

OVERVIEW §9 принимал «содержимое меддокументов уходит AI-провайдерам». Посредник добавляет в цепочку **ещё одно звено** без внятной политики обработки данных и DPA: документы семьи проходят через инфраструктуру RouterAI. Владелец осознанно принимает это для POC. Флаг MVP из §9 (сменный провайдер / локальная модель) становится ещё актуальнее.

## Последствия

- (+) Разблокирован этап 4 целиком, включая живую проверку качества на реальных документах.
- (+) Адаптер универсален: смена агрегатора или переезд на официальный API — конфиг, не код.
- (−) Стабильность и актуальность моделей — на совести посредника; SLA нет; structured output/tool use через прослойку не гарантированы — адаптер обязан уметь «JSON в тексте + валидация + повтор».
- (−) Цены с маржой (для наших объёмов несущественно).
- Проверка vision живым запросом — первый шаг T4.2, до какой-либо ставки на провайдера.
