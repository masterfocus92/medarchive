# ADR-009 — Данные seed вне git: код версионируется, данные — нет

## Статус

Принято владельцем 13.07.2026 (обсуждение нарезки этапа 2, уточнено при плане T2.1-BE).

## Контекст

Seed-скрипт создаёт семейное пространство с реальными персональными данными: ФИО, даты рождения, email'ы, пароли. Репозиторий живёт на GitHub — утечка этих данных недопустима (требование владельца: «не должен никуда протечь»). Первоначальный вариант — скрыть из git весь скрипт — обсуждён и заменён компромиссом: невersionируемый скрипт тихо ломается при изменении моделей.

## Решение

- **Код** (`app/seed.py`) — в git; не содержит ни одной буквы реальных данных.
- **Данные** — только в `.env.seed` (в `.gitignore`); класс `SeedSettings` в `app/config.py` (чтение окружения — в одном модуле, ADR-001).
- В git — обезличенный шаблон `.env.seed.example`; пароли в шаблоне пустые, запуск «как есть» падает с перечнем незаполненных полей до обращения к БД.
- Тесты используют собственные тестовые данные — реальный `.env.seed` им не нужен.

## Последствия

- (+) Скрипт версионируется и покрыт тестами — изменение моделей ломает его громко, в CI-прогоне.
- (+) Утечка данных структурно невозможна: им негде появиться в репозитории.
- (−) `.env.seed` существует только локально: новая машина = заполнить шаблон заново (или перенести файл руками).
- Правило распространяется вперёд: любые будущие реальные данные семьи в код и git не попадают.
