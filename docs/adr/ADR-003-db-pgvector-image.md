# ADR-003 — БД разработки: образ pgvector/pgvector:pg16, named volume, креды через `.env`

## Статус

Принято 13.07.2026 (основа compose-файла — владельца; доведение до критериев T1.1 — по одобренному плану).

## Контекст

Стек фиксирует PostgreSQL 16 + pgvector в docker-compose. Стандартный образ `postgres:16` расширения vector не содержит; собирать свой образ для POC — лишняя сущность. Критерии T1.1: данные переживают пересоздание контейнера, окружение поднимается без ручной настройки.

## Решение

- Образ — `pgvector/pgvector:pg16` (официальный образ проекта pgvector, расширение внутри).
- Данные — в named volume `pgdata` (не anonymous).
- Креды и имя БД — `${POSTGRES_*:-medcard}`: дефолт `medcard` совпадает с `.env.example`, `.env` позволяет переопределить.
- Healthcheck на `pg_isready` — готовность БД видна compose и будущим зависимым сервисам.
- Вектор-колонки на этом этапе не создаются — задача compose лишь в доступности `CREATE EXTENSION vector` (сами эмбеддинги — этап 7).

## Последствия

- (+) `docker compose up -d` работает на чистом клоне без настройки.
- (−) Смена кред **после** первой инициализации не подействует: Postgres читает `POSTGRES_*` только при создании кластера. Смена требует пересоздания volume (`docker compose down -v` — с потерей данных).
- Тег `pg16` плавающий (минорные обновления Postgres приезжают сами); для VPS-деплоя (этап 8) версию стоит зафиксировать точнее.
