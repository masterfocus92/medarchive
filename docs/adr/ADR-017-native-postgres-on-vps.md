# ADR-017 — На VPS PostgreSQL ставится нативно; docker остаётся для разработки

## Статус

Принято владельцем 15.07.2026. Уточняет ADR-003 (докер-образ pgvector — для локальной разработки; ADR-003 в силе).

## Контекст

VPS — 1 core / 1 ГБ RAM, на нём должны жить два контура приложения (prod+stg), существующая PWA владельца и Caddy. Docker-демон на такой машине съедает ощутимую долю памяти, не давая взамен ничего: контейнер БД на проде был бы один и вечный.

## Решение

- На VPS PostgreSQL 16 ставится **нативно** из apt-репозитория PGDG, расширение — пакетом `postgresql-16-pgvector`. Версия — та же, что в dev-образе `pgvector/pgvector:pg16` (ADR-003): среда разработки и прод согласованы по версии, отличается только способ упаковки.
- Одна инсталляция, две БД (`medcard_prod`, `medcard_stg`) с отдельными ролями; лёгкий тюнинг под 1 ГБ (умеренный `shared_buffers`, ограниченные `max_connections`).
- Swap 2 ГБ на VPS — страховка от OOM.

## Последствия

- Docker на VPS не нужен вовсе — экономия памяти и одним демоном меньше.
- Обновление мажорной версии PG — руками через `pg_upgrade`/dump-restore (для POC приемлемо; событие редкое).
- Инструкция установки — в тикете инфраструктуры и `RELEASE.md`; dev-окружение не меняется.
