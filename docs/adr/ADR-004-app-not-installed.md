# ADR-004 — Пакет `app` не устанавливается в venv

## Статус

Принято 13.07.2026 (решение по ходу T1.1: тесты не видели пакет `app`; зафиксировано в отчёте, принятом владельцем).

## Контекст

Тесты и приложение импортируют пакет `app`. Варианты: собирать проект как устанавливаемый пакет (build-backend, editable install при `uv sync`) — либо импортировать прямо из рабочей копии. Проект — POC-приложение, не библиотека: его никто не ставит через pip.

## Решение

Проект не собирается и не устанавливается. Импорт `app` — из корня рабочей копии; для pytest корень добавлен в `sys.path` настройкой `pythonpath = ["."]` в `pyproject.toml`.

## Последствия

- (+) Нет build-backend и packaging-конфигурации — минус целый класс забот.
- (−) Все процессы (uvicorn, alembic, pytest) запускаются из корня проекта — иначе импорт сломается.
- Если деплой (этап 8) потребует установки как пакета — вернуться к этому ADR через владельца.
