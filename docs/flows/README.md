# Пользовательские потоки — реестр

Детальные пользовательские пути: экраны, кнопки, события, ветвления, задействованные компоненты, схемы PlantUML. Поток — источник истины для нарезки тикетов и реализации соответствующих этапов: тикет и код сверяются с потоком, расхождение — дефект тикета/кода или осознанное изменение потока через владельца.

| Поток | Покрывает | Статус |
|---|---|---|
| [record-creation.md](record-creation.md) | Создание записи: камера/файлы, сохранение, AI-разбор, проверка, подтверждение (этапы 3–4, стыки с 5 и 7) | утверждён владельцем 14.07.2026 |
| [record-viewing.md](record-viewing.md) | Просмотр: лента по профилю (2 сортировки), карточка записи, листание страниц, оригиналы (этап 5) | утверждён владельцем 15.07.2026 |
| [record-editing-deletion.md](record-editing-deletion.md) | Правка (экран проверки в режиме правки) и удаление (soft + каскадная дисциплина, подтверждение «листом») — этап 6, стык с Э7 | утверждён владельцем 15.07.2026 |
| [search.md](search.md) | Поиск-чат: вопрос → retrieval (pgvector) → генеративный ответ с обязательными источниками; индексация при подтверждении/правке, чистка при удалении (этап 7) | утверждён владельцем 19.07.2026 |

Будущих кандидатов нет — все продуктовые потоки написаны.
